Binary Indexed Tree

2020-06-30 Algorithms PV:

Definition

  1. A data structure that can efficiently update element and calculate prefix sums in a table of numbers
  2. It can solve the problems below efficiently
    • update(idx, delta): add num at the number which at position idx
    • prefixSum(idx): get the sum of numbers which start from position 1 to idx(include idx)
    • rangeSum(from_idx, to_idx): get the sum of all the numbers which is from from_idx to to_idx

Thoughts about Construction

  1. Binary Indexed Tree store the element in a logical way according to the binary of number
  2. Sum in the Binary Indexed Tree:
    • Given the position i(we want to get the sum from position 1 to i), like 13
    • 13 = 2^3 + 2^2 + 2^0 => prefixSum(13) = RANGE(1, 8) + RANGE(9, 12) + RANGE(13, 13)
    • Note: RANGE(x, y) represent the sum of array starting from postion x to postion y
      • First level, fill from the first number in the array. length is the sum of 2^m(m is from 0 to infinite) numbers
        for example 2^0, 2^1, 2^2, 2^3. stop here because 2^4 exceeds the length of array
      • Second level. fill from 3(the first hole). length is the sum of 2^m(m is from 0 to infinite) numbers — always start from hole, to the end of array
      • Third level is same as previous one
    • contruct a binary indexed array, we call it BIT, for our coonvience, we index it starting from 1
    • prefixSum(13) = prefixSum(0b0001101) = BIT[13] + BIT[12] + BIT[8]

How to get prefix sum or range sum

Way to find the parent node of current node

  1. Observation From Below
    prefixSum(13) = prefixSum(0b0001101)
    = BIT[13] + BIT[12] + BIT[8]
    = BIT[0b00001101] + BIT[0b00001100] + BIT[0b00001000]
  2. Move from bottom to Up, actually is a process to flip last digit 1 to 0, for example x = 13
    • x = 13 = 0b00001101
    • -x = -13 = 0b11110011
    • x & (-x) = 0b00000001
    • x - (x & (-x)) = 0b00001100

Update elment in the array

  1. For example, update(5,2), we want to add 2 to the number at postion 5, the process of it would like:
  2. we can see that, the postition needed to update is the binary of original position plus 1 to the last 1 in the binary representation, for example x = 5
    • x = 5 = 0b00000101
    • -x = -5 = 0b11111011
    • x & (-x) = 0b00000001
    • x + (x & (-x)) = 0b00000110

Construction of Binary Indexed Tree

Method 1

  1. Iinite an array with all 0,
  2. Do the update(i, delta) for every item –> O(nlgn)

Method 2

Input is an array list with length n

  • Initianize an array bit with length n+1
  • Put the numbers of list to bit array from position 1 to n
  • for index i from 1 to n
    • j = i + (i & -i)，if j < n + 1，then bit[j] = bit[j] + bit[i]

Conclusion

  1. 如何确定某个位置到底是有几个数组成的呢
  2. 原来是根据坐标的最低位 Low Bit 来决定的，所谓的最低位，就是二进制数的最右边的一个1开始，加上后面的0(如果有的话)组成的数字

Notes

  • (i & -i): trick to extract the lowest set bit of i
