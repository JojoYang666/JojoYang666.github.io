React LifeCycle

2020-06-25 Front EndReact PV:

Lifecycle Diagram

Updating

componentDidUpdate

  1. invoked immediately after updating occurs, is not called for the initial render
  2. A good place to do network requests as long as you compare the current props to previous props(e.g. a network request may not be necessary if the props have not changed)
  3. May call setState() immediately in componentDidUpdate(), but nore that it must be wrapped in a condition, otherwise will cause a infinite loop
  4. If try to “mirror” some state to a prop coming from above, consider using the prop directly instead

shouldComponentUpdate

  1. Let react know if a component’s output is not affected by the current change in state or props
  2. Default behavior is to re-render on every state change, and in the vast majority of cases you should rely on the default behavior
  3. Invoked before rendering when new props or state are being received. Defaults to true
  4. Exists as a performance optimization, do not rely on it to “prevent” a rendering, as this can lead bugs
    • If want to prevent a rendering, can use PureComponent
    • PureComponent performs a shallow comparison of props and state, and reduces the chance that you’ll skip a necessary update
  5. compare this.props with nextProps and this.state with nextState and return false to tell React the update can be skipped
    • Noted that returning false does not prevent child components from rerendering when their states changes
  6. Not recommend doing deap equality checks or using JSON.stringify() in shouldComponentUpdate(), it is every inefficient and will harm the performanced

React.PureComponent

  1. similar to React.Component. The difference between them is:

    • React.Component does not implement shouldComponentUpdate()
    • React.PureComponent implements it with a shallow prop and state comparison

  2. React.PureComponent’s shouldComponentUpdate only shallowly compares the object

    • if contains complex data structer, may introduce false-negative for deeper diff

  3. React.PureComponent’s shouldComponentUpdate skips prop updates for the whole component subtree

    • Make sure all the children components are also pure
