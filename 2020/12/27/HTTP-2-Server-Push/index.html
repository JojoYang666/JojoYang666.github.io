HTTP/2 & Server Push

2020-12-27 Web Development PV:

Problems with HTTP/1

  1. causes huge performance blocks
  2. all the requests happen one after the other, and each request requires a separate TCP connection to be opened.
  3. adds up in the website’s loading time.

Solution: HTTP/2

  1. HTTP/2 aims to load as much as possible in as few round trips as possible.
  2. It also does so over one single TCP connection, which reduces connection establishment and termination overhead
  3. it uses HPACK header compression, which also provides a performance boost.

    Server push

    getting the server to send all the necessary files in one round trip over one TCP connection without the client even firing off a request.

Multiplexing

multiple files can be sent, or ‘multiplexed,’ in any order over one TCP connection from the server to the client.
