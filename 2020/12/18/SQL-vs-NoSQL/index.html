SQL vs. NoSQL

2020-12-18 System Design PV:

SQL

  1. most popular relational databases
    • MySQL
    • Oracle
    • MS SQL Server
    • SQLite
    • Postgres
    • MariaDB

NoSQL

the most common types of NoSQL

Key-Value Stores

  1. The ‘key’ is an attribute name which is linked to a ‘value’.
  2. key-value db:
    • Redis
    • Voldemort
    • Dynamo

Document Databases:

  1. data is stored in documents
  2. these documents are grouped together in collections
  3. Each document can have an entirely different structure
  4. Document databases:
    • CouchDB
    • MongoDB

Wide-Column Databases

  1. have column families, which are containers for rows
    • not need to know all the columns up front
    • each row doesn’t have to have the same number of columns.
  2. best suited for analyzing large datasets - big names include Cassandra and HBase.

Graph Databases

  1. used to store data whose relations are best represented in a graph
  2. Data is saved in graph structures
    • with nodes (entities), properties (information about the entities)
    • lines (connections between the entities)
  3. graph:
    • Neo4J
    • InfiniteGraph

High level differences between SQL and NoSQL

Storage

  1. SQL stores data in tables
  2. NoSQL databases have different data storage models

Schema

  1. SQL

    • columns must be decided and chosen before data entry
    • each row must have data for each column
    • schema can be altered later, but it involves modifying the whole database and going offline.

  2. nosql:

    • schemas are dynamic
    • Columns can be added on the fly
    • each ‘row’ (or equivalent) doesn’t have to contain data for each ‘column.’

Querying

  1. sql
    • use SQL (structured query language) for defining and manipulating the data
  2. nosql:
    • queries are focused on a collection of documents.
    • also called UnQL (Unstructured Query Language).
    • Different databases have different syntax for using UnQL.

Scalability

  1. SQL

    • vertically scalable –> expensive
    • possible to scale a relational database across multiple servers, but this is a challenging and time-consuming process.

  2. nosql

    • horizontally scalable
    • cheap commodity hardware or cloud instances
    • more cost-effective than vertical scaling

Reliability or ACID Compliancy (Atomicity, Consistency, Isolation, Durability)

  1. sql –> ACID compliant
  2. Most of the NoSQL solutions sacrifice ACID compliance for performance and scalability.

SQL VS. NoSQL - Which one to use?

Reasons to use SQL database

  1. need to ensure ACID compliance –> for many e-commerce and financial applications
  2. data is structured and unchanging —> business is not experiencing massive growth

Reasons to use NoSQL database

  1. Storing large volumes of data that often have little to no structures
  2. Making the most of cloud computing and storage
    • d NoSQL databases like Cassandra are designed to be scaled across multiple data centers out of the box, without a lot of headaches.
  3. Rapid development.
    • system which require making frequent updates to the data structure without a lot of downtime between versions, a relational database will slow you down.
